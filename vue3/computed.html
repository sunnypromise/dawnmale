# 计算属性

使用场景：在模板中写太多逻辑避免模板变得臃肿,使用计算属性来描述依赖响应式状态的复杂逻辑

computed() 方法期望接收一个 getter 函数，返回值为一个计算属性 ref。可以通过computed.value 访问计算结果,计算属性 ref 也会在模板中自动解包，因此在模板表达式中引用时无需添加 .value。 ex:

    <script setup lang="ts">
    import { computed, reactive } from "vue";
    const data = reactive({
        name: "dawnmale",
        list: ["cool", "beautiful"],
    });
    const peopleDep = computed(()=>{
        return data.list.length < 0 ? data.name : "default name";
    })
    </script>
    <template>
        <global-component />
        <div>{{ peopleDep }}</div>
    </template>

计算属性值会基于其响应式依赖被缓存 也就以为着必须要接受一个依赖一个响应式的ref，当ref更新时，计算属性才会重新计算

下面的计算属性永远不会更新，因为 Date.now() 并不是一个响应式依赖

    const now = computed(() => Date.now())

# 可写计算属性

    <script setup lang="ts">
    import { computed, ref } from "vue";
    const firstName = ref("default name");
    const lastName = ref("last name");
    const fullName = computed({
    // 获取值
    get() {
        return firstName.value + " " + lastName.value;
    },
    // 设置值
    set(newValue) {
        [firstName.value, lastName.value] = newValue.split(" ");
        // 使用分割把传入的string转换成数组['dawn','male']
    },
    });
    const handleChangeValue = () => {
      fullName.value = "dawn male";
      // 更改fullName，setter 会被调用 依赖ref会随之更新
    };
    </script>

    <template>
    <div>
        {{ fullName }}
        <button @click="handleChangeValue">chang computed value</button>
    </div>
    </template>

# 注意

计算属性的 getter 应只做计算而没有任何其他的副作用，不要在 getter 中做异步请求或者更改 DOM!!!
